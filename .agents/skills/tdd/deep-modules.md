# 深模块（Deep Modules）

出自《A Philosophy of Software Design》：

**深模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│   Small Interface   │  ← 方法少、参数简单
├─────────────────────┤
│                     │
│                     │
│  Deep Implementation│  ← 复杂逻辑藏在内部
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现（应避免）

```
┌─────────────────────────────────┐
│       Large Interface           │  ← 方法多、参数复杂
├─────────────────────────────────┤
│  Thin Implementation            │  ← 多为透传
└─────────────────────────────────┘
```

设计接口时可问：

- 能否减少方法数量？
- 能否简化参数？
- 能否把更多复杂度藏在内部？
