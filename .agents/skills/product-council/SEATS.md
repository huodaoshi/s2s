# 议会席配置（可编辑）

本文件是**默认席位的菜单**。运行 `product-council` 时，Agent 会先展示此列表，请你确认、删减或追加；**不必改 SKILL.md**。

也可为单次项目在 `.scratch/<feature-slug>/council-seats.md` 放覆盖版；若存在则**优先**读项目版。

---

## 推进席（Advocate）

每轮 **1 顶帽**；按议题轮换，不必每轮问用户选谁。

| 启用 | 标签（可改） | 审视重点 |
| ---- | ------------ | -------- |
| yes | 产品设计师 | 用户问题、场景、MVP 范围、体验路径 |
| yes | 技术负责人 | 默认技术栈、交付风险、运维与扩展 |
| no | 增长 / 商业 | 获客、定价、商业模式、竞品差异化 |

## 质疑席（Skeptic）

每轮 **1～3 席**；每席 **最多 2 条**可证伪反驳 + 1 条验证实验建议。标签是**视角名**，不是固定职业。

| 启用 | 标签（可改） | 审视重点 |
| ---- | ------------ | -------- |
| yes | 市场与证据 | 需求是否成立、竞品、数据来源 |
| yes | 技术与交付 | 可行性、复杂度、可靠性、成本 |
| yes | 范围与动机 | scope creep、是否在做「自己想做的」、伦理红线 |
| no | 合规与风险 | 隐私、牌照、平台政策 |

---

## 编辑说明

- 改 **启用** 为 `yes` / `no`，或整行删掉。
- **标签** 随意改（如「资深工程师」「红队」）；Agent 按「审视重点」列发言。
- 想加席：复制一行表格行即可。
- 入场时可直接在对话里说：「质疑只要市场+技术，推进默认产品设计师」——Agent 按你的口头配置覆盖上表，无需先改文件。
